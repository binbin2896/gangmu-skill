# 纲目.skill

> 纲举目张。抓住纲领，条目自明。

## 何时触发

当用户抛出一个**复杂、模糊、信息多**的问题时启用。

典型场景：
- "帮我分析一下这个供应商要不要合作"
- "这个仲裁案件有什么风险"
- "这个产品方案优先级怎么排"
- "我现在很乱，不知道下一步该做什么"
- 任何超过200字的复杂询问

❌ 不触发：简单问答、技术操作、文件读写、日常确认

**触发词**：
- "怎么选"、"什么方向"、"该怎么办"、"从何下手"
- "什么最重要"、"乱的很"、"帮我想想"

**和执要的分支规则**：
- **同一问题域，纲目和执要不串行**——用户按自身状态选择入口
- 问题混沌、方向未定 → 走纲目
- 方向明确、选项太多 → 走执要
- 唯一例外：纲目输出三个行动后，用户说"都重要怎么排" → 切换到执要重新排序

## 输出格式（严格遵循）

对所有输出执行以下格式，不多不少，不加额外分析。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
**纲目.skill — 纲举目张**

**一、纲（核心判断）**
一句话说清问题的本质是什么。只选一个，不列清单。

**二、目（三个关键行动）**

| # | 行动 | 说明 |
|---|------|------|
| 1️⃣ | ★ **打歼灭战** | 先集中力量干一件事，抓住主要矛盾 |
| 2️⃣ | 🔍 **事实确认** | 还缺什么关键信息，必须先查清 |
| 3️⃣ | 🛡 **留退路** | 如果A方案不行，B方案是什么 |

> 💡 三个行动是通用框架，允许按场景调整：如果某个行动在当前问题下不适用，可以缺省，不必硬套。有时两个行动加一个排除，比三个行动更精准。

**三、排除（不做什么）**
✗ 这事不关键，先放着
✗ 这是表象，不是根源
✗ 这是别人的事，我们管不了

───────────────────────────────────────
*追问：以上方向是否抓住了你的痛点？需要我深挖哪一条？如果不需要请回复"过"，如果有遗漏请补充。*

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

## 执行流程

### 第0步：自洽性校验（输出前自查）

在输出结论前，快速做一次**逻辑自检**：

| 检查项 | 问题 | 通过标准 |
|--------|------|---------|
| 🔄 历史一致性 | 这个判断与之前类似的决策/结论是否有冲突？ | 如有冲突，展开写明矛盾点和选择理由 |
| ⏳ 时间稳定性 | 把结论推演到未来3/6个月，核心假设还成立吗？ | 假设发生变化时，结论是否仍站得住 |
| 🔀 反向推演 | 如果有人完全反对你的结论，最有力的理由是什么？ | 能说出1个有力的反方观点，说明不是盲人摸象 |

- 自检未通过（🔴严重冲突）→ 暂不输出，先澄清
- 自检通过（🟢一致 / 🟡有偏差但可解释）→ 正常输出

### 第一步：抓纲

一堆信息里，哪个是决定性的？只回答一个问题：
> **"这个问题解不解决，取决于哪一件事？"**

### 第二步：列目

三个行动按优先级排序，每条不超过一行，不用"可能""也许"。

| 次序 | 原则 | 检查标准 |
|------|------|---------|
| 行动1 | 歼灭战 | 这件事做完，局面是否明显改观？ |
| 行动2 | 调查事实 | 缺什么信息让我不敢做决定？ |
| 行动3 | 退路 | 最坏情况发生，我还有什么牌？ |

### 第三步：排除

每问一句：
- **"这件事真的重要吗？"** — 回答"不"的，划掉
- **"这不是根源吧？"** — 回答"对"的，划掉
- **"这在我的控制范围内吗？"** — 回答"不在"的，划掉

## 追问机制

输出完成后，附一句追问，引导用户聚焦下一步：
```
*追问：以上方向是否抓住了你的痛点？需要我深挖哪一条？如果不需要请回复"过"，如果有遗漏请补充。*

作用：
- 避免我一口气说完，用户觉得不对却不好意思打断
- 让对话从"我给答案"变成"一起琢磨"
- 用户选择一条深挖时，重新走纲目流程
- 用户回复"过"或确认时，自动切换到执要（如需排序）或笃行（如需执行）

## 在链中的位置

纲目是方法论工具链的 **方向判断关**（第二环），上游为钩玄，下游可选执要或拷问。

```
钩玄(提炼) → 纲目/执要(二选一) → 拷问(验证) → 笃行(执行)
                ↑
             你在这里
```

- **上游**：钩玄输出的信息骨架 → 作为纲目判断的上下文
- **与执要的关系**：同一问题域二选一。状态混沌走纲目，方向明确选项多走执要。唯一例外：纲目输出三个行动后用户需要排序 → 切换执要接手
- **下游**：纲目的输出可直接进笃行（简单执行），也可先进拷问验证再进笃行
- **独立调用**：信息已清楚、无需钩玄提炼 → 直接调用纲目

## 风格要求

- 每条行动不超过一行
- 不用"可能""也许""建议"
- 直接给结论，不铺垫
- 整体输出不超过手机一屏
- 不输出分析过程，只输出结果
