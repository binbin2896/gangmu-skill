# 纲目.skill — 复杂问题拆解框架

> 纲举目张。抓住纲领，条目自明。

## 一个核心判断，三个关键行动，一组排除项

不分析，不铺陈，只给结论和下一步。适用于供应商决策、案件风险评估、方向选择等复杂模糊场景。

## 输出格式

```
一、纲（核心判断）
一句话说清问题的本质

二、目（三个关键行动）
1️⃣ 打歼灭战 — 先集中力量干一件事
2️⃣ 事实确认 — 缺什么关键信息必须先查清
3️⃣ 留退路 — 如果A方案不行，B方案是什么

三、排除（不做什么）
✗ 这事不关键，先放着
✗ 这是表象，不是根源
✗ 这是别人的事，管不了
```

## 核心逻辑

| 步骤 | 原则 | 检查标准 |
|------|------|---------|
| 抓纲 | 抓住决定性因素 | 这个问题解不解决，取决于哪一件事？ |
| 列目 | 三个行动按优先级 | 歼灭战 / 调查事实 / 留退路 |
| 排除 | 三问筛掉不重要的 | 重要？根源？可控？ |

## 适用场景

- 供应商/合作方要不要做
- 仲裁案件风险评估
- 多方信息下的方向选择
- 任何信息超200字的复杂询问
